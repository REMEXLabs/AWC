Requirements

The core library is based on Web Components, which is a term of the use of the following noval technologies. The following list shows the different technologies and approaches.

  • HTML Templates

    Method of declaring a portion of reusable markup that is parsed but not rendered until cloned.

  • Custom Elements

    Method of defining and using new types of DOM elements in a document.

  • HTML Imports

    Method of including and reusing HTML documents in other HTML documents.

  • Shadow DOM

    Method of establishing and maintaining functional boundaries between DOM trees and how these trees interact with each other within a document, thus enabling better functional encapsulation within the DOM.

Browser Support

Web Components

The Web Components is the future technology of modern websites. But the implementation of the required features is still under development. So we recommend using the newest stable Google Chrome browser.

ECMAScript 6

The library componenets are written in ECMAScript 6, which is the highest level of JavaScript today. But in general the browser support of the related new features are limited. Because of that issue we use the JavaScript transpiler Babel to enhance the compatibility.

Build Process

Requirements

Installation

The file package.json lists all necessary dependencies for a build. Execute the following command to install all packages.

$ npm install

Note: Be patient, the download and installation process takes a moment.

Process

Run the build process by typing:

$ gulp build

After that you can start a local webserver (127.0.0.1:3000) and try all examples:

$ npm start

Browser

Start the Google Canary with following necessary flags:

$ <Application> --enable-blink-features=CustomElementsV1,ShadowDOMV1
